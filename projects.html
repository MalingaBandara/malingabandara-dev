Currency Conversion and Exchange API with Microservices



Project Description:

The Currency Conversion and Exchange API project is designed to facilitate seamless currency conversion and exchange operations through a set of coordinated microservices. Each service is built using Spring Boot, with Docker used for containerization to ensure that the microservices are easily deployable and scalable across different environments. This architecture not only enhances the system's resilience and maintainability but also supports dynamic scaling according to demand.

The system's distributed nature is managed through Eureka, which enables service discovery, while Zipkin provides capabilities for distributed tracing, crucial for monitoring and troubleshooting in a microservices architecture. Communication between services is handled asynchronously via RabbitMQ, enhancing the system's efficiency and response times.

Business Case:

This application is pivotal for financial platforms requiring real-time currency data to serve global markets. It supports high-volume data processing with minimal latency, making it an essential tool for financial forecasting, analysis, and transaction services.

Technologies Used:

  • Spring Boot: Utilized for rapid development of microservices.
  • Microservices Architecture: Applied to achieve scalability, resilience, and maintainability.
  • Cloud Computing: Deployed on cloud platforms for enhanced scalability and accessibility.
  • Docker: Used for containerizing the services to ensure consistency across different deployment environments.
  • Eureka: Implemented for service discovery to manage microservices efficiently.
  • Zipkin: Integrated for distributed tracing to monitor and troubleshoot microservices.
  • H2 Database: Employed as a lightweight in-memory database during development for storing exchange rates.
  • RabbitMQ: Utilized for asynchronous communication between microservices.
  • Resilience4j: Incorporated for fault tolerance to enhance the robustness of the service.

Core Features:

  • Currency Conversion Microservice: Converts currencies based on real-time exchange rates.
  • Currency Exchange Microservice: Handles the actual exchange of currencies using up-to-date rates.
  • Naming Server Microservice: Facilitates the discovery and registration of microservices using Eureka.
  • API Gateway Microservice: Acts as the central point of entry, routing requests to the appropriate microservices.

Augmented Reality Spectacles Fit-On App



Project Description:

The Augmented Reality Spectacles Fit-On app revolutionizes the eyewear shopping experience by allowing users to virtually try on different spectacle frames. The app features a user-friendly welcome screen where users can select from three different frame styles. Once a style is selected, the app transitions to the AR Fit-On screen, which overlays the selected frame onto the user’s face in real-time using ARKit.

The application leverages Unity to ensure that the frames are rendered with high-quality textures and adjusted precisely to the user’s face. ARKit’s face-tracking capabilities enable smooth and accurate placement, providing a lifelike experience as the user moves or changes angles. This technology allows users to see exactly how the frames will look on them before making a purchase, reducing the need for physical try-ons.

Business Case:

This app is a valuable tool for eyewear retailers looking to offer a modern, technology-driven shopping experience. By integrating augmented reality, the app bridges the gap between online and in-store shopping, enabling customers to confidently choose the right frames from the comfort of their homes. The app also provides a significant marketing advantage by positioning the brand as innovative and customer-centric.

Technologies Used:

  • ARKit: Utilized for augmented reality capabilities, particularly for real-time face tracking and overlaying 3D spectacle frames.
  • Unity: Employed for rendering and adjusting 3D spectacle frames to match the user's face in real-time.
  • Blender: Used for creating detailed 3D models of spectacle frames.
  • Swift: The primary programming language for the iOS app development.
  • Xcode: Used as the integrated development environment for building, testing, and deploying the app.

Core Features:

  • Welcome Screen with Frame Selection: The app launches with a clean and simple welcome screen that offers three different spectacle frame options. Users can select a frame, which then loads the AR Fit-On screen with the chosen frame enabled for real-time virtual try-on.
  • Interactive AR Experience: The AR Fit-On screen utilizes ARKit’s advanced face-tracking technology to accurately position the selected spectacle frames on the user's face. The face-tracking works seamlessly, providing a realistic and fluid experience as the user moves or changes angles.
  • User-Friendly Interface: The UI is intuitive and easy to navigate, ensuring users can switch between frames and explore the virtual try-on feature with minimal effort. The transition from the welcome screen to the AR experience is smooth and optimized for performance.

The SLBFE system is designed to manage the complex interactions between citizens seeking employment abroad, foreign companies, and the Sri Lankan Bureau of Foreign Employment. It provides a streamlined process for handling citizen profiles, qualification assessments, and communication between relevant parties. The system is built with a user-centric design and offers customized features and interfaces based on user roles, such as citizens, foreign companies, and bureau officials.

Business Case:

This system is crucial for organizations involved in managing foreign employment processes, providing a centralized solution to handle citizen profiles, job applications, and communications. It offers streamlined workflows that reduce administrative overhead and enhance the coordination between stakeholders, improving the overall experience for citizens and companies alike.
